The **KYI Widget** is a drop-in React component that renders Bluprynt's verified-issuer view inside your site. It's distributed as `@bluprynt/kyi-widget-sdk` on npm.

## Prerequisites

<Check>You have an access token from Bluprynt — contact [partners@bluprynt.com](mailto:partners@bluprynt.com) for one.</Check>
<Check>A Node.js backend to issue short-lived JWTs to your frontend. (The SDK ships a server helper, but you can also sign yourself.)</Check>
<Check>Your frontend is registered as an allowed embed origin with Bluprynt — the `ALLOWED_EMBED_ORIGINS` CSP gate will block unregistered origins.</Check>

## Generate an access token (server-side)

```typescript server.ts
import { generateToken } from '@bluprynt/kyi-widget-sdk/server';

app.get('/kyi-token', async (req, res) => {
  const token = await generateToken({
    secretKey: process.env.BLUPRYNT_SECRET_KEY!,
    subject: req.user.id,           // identifies the end user
    walletAddress: req.query.wallet as string,
    expiresInSeconds: 300,          // 5 minutes is typical
  });
  res.json({ token });
});
```

<Warning>
  **Never expose `BLUPRYNT_SECRET_KEY` to your browser.** Always mint tokens on your server.
</Warning>

## Mount the widget (client-side)

```typescript App.tsx
import { BlupryntKyiWidget } from '@bluprynt/kyi-widget-sdk';

function IssuerPage({ walletAddress }: { walletAddress: string }) {
  const [token, setToken] = useState<string | null>(null);

  useEffect(() => {
    fetch(`/kyi-token?wallet=${walletAddress}`)
      .then(r => r.json())
      .then(d => setToken(d.token));
  }, [walletAddress]);

  if (!token) return <Skeleton />;

  return <BlupryntKyiWidget accessToken={token} />;
}
```

## Customization

The widget inherits your site's base font and supports a theme prop:

```typescript
<BlupryntKyiWidget
  accessToken={token}
  theme="light"       // "light" | "dark" | "auto"
  primaryColor="#007BFF"
/>
```

For deeper customization, see the [SDK repo](https://github.com/blupryntco/kyi-widget-sdk).

## Troubleshooting

<AccordionGroup>
  <Accordion title="Widget shows 'Unauthorized'">
    Your token is expired or invalid. Token TTLs should be short (5–10 min); mint a fresh one per session.
  </Accordion>
  <Accordion title="Widget is blocked by CSP">
    Your origin isn't in Bluprynt's `ALLOWED_EMBED_ORIGINS` allowlist. Email [partners@bluprynt.com](mailto:partners@bluprynt.com) with the exact origin (`https://example.com`).
  </Accordion>
  <Accordion title="The widget renders but says 'No issuer found'">
    The wallet address isn't associated with a KYI-verified issuer. This is the correct behavior for non-verified assets; you can either hide the widget or show a "Not verified" message.
  </Accordion>
</AccordionGroup>
