**MiCA Checker** is Bluprynt's dedicated compliance workflow for issuers subject to the [EU Markets in Crypto-Assets Regulation (MiCA)](https://eur-lex.europa.eu/eli/reg/2023/1114/oj). It streamlines the five filing obligations most MiCA issuers face into a single guided authoring flow.

## What it covers

## Who should use it

- Issuers based in the EU launching tokens subject to MiCA.
- Non-EU issuers with EU distribution that triggers MiCA obligations.
- Legal counsel filing on behalf of token issuers.

## How it differs from standard Smart Disclosures

| | Standard Smart Disclosures | MiCA Checker |
|---|---|---|
| Jurisdiction coverage | Multi-jurisdiction | EU MiCA only |
| Signup flow | `/signup` | `/signup-mica` (streamlined) |
| AI assistant | General OpenAI Assistant | Dedicated `MICA_CHECKER_ASSISTANT_ID` tuned for MiCA |
| Form templates | Broad | Pre-loaded with MiCA-specific field sets |
| Classification | Generic | Interactive ART / EMT / utility classifier |
| Regulator templates | Generic | ESMA + NCA specific |

## Reserves attestations (ART / EMT issuers)

Asset-referenced token and e-money token issuers have recurring reserves-attestation obligations. MiCA Checker automates this:

- **Monthly proof-of-collateral attestations** written on-chain using the `proof-of-collateral` attestation type.
- **Audit integration** — link an approved audit firm's report as the attestation payload.
- **Public verification** — any counterparty can verify reserves via the on-chain attestation without calling Bluprynt.
