The **Bluprynt Public API** exposes KYI-verified asset metadata and verification badges. It powers the verification surfaces partners build to answer "is this token issued by who it claims to be issued by?" — exchange listings, wallet warnings, explorer details, dApp gating, and trust UI.

<Tip>
  The interactive playground for this API is at [integrations.bluprynt.com](https://integrations.bluprynt.com). The same OpenAPI spec is rendered into these docs and lives at [`/api-reference/openapi.json`](/api-reference/openapi.json).
</Tip>

## Authentication

Every Asset endpoint requires an API key sent in the `Authorization` header. Badge requests are public. See the [Authentication](/api-reference/authentication) page for the full pattern.

```bash
curl https://integrations.bluprynt.com/assets \
  -H "Authorization: YOUR_API_KEY"
```

## Endpoints at a glance

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/assets` | [List KYI-verified assets](/api-reference/endpoints/list-assets) (paginated). |
| `GET` | `/assets/{chain}/{account_address}` | [Get an asset by chain and address](/api-reference/endpoints/get-asset). |
| `GET` | `/assets/{namespace}/{reference}/{account_address}` | [Get an asset by CAIP-10 account ID](/api-reference/endpoints/get-asset-by-caip-10). |
| `GET` | `/api/v1/badges/{credential_type}` | [Get a verification badge image](/api-reference/endpoints/get-badge) (no auth). |

## Supported chains

Asset endpoints accept the following chain identifiers:

`ethereum` · `base` · `solana` · `plume` · `avalanche` · `sepolia`

CAIP-10 lookups support any namespace/reference combination Bluprynt has assets for — see the [CAIP-10 lookup endpoint](/api-reference/endpoints/get-asset-by-caip-10) for the URL-segment encoding rule.

## Response format

All Asset responses are JSON. Successful asset lookups return an `AssetInfo` object with two nested objects: `token_details` (chain, address, name, ticker, controlling authorities, image) and `business_details` (issuer DID, KYB status, business name and description). Listing returns the same `AssetInfo` shape inside a paginated `data` / `meta` envelope.

Badge responses are binary image data with `Cache-Control: public, max-age=31536000, immutable`.

## Status

The API is **production**. Schema changes are additive; deprecations are announced via email and via the [changelog](https://github.com/blupryntco/docs/blob/main/CHANGELOG.md) before removal.
