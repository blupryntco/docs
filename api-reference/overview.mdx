## Base URL

| Environment | URL |
|---|---|
| Production | `https://api.bluprynt.com` |
| Staging | `https://api.staging.bluprynt.com` |

## Versioning

Every endpoint is prefixed with a version: `/v1/...`. Breaking changes introduce a new major version (`/v2`) and the previous version remains available for at least 12 months. Non-breaking additions (new optional fields, new endpoints) go into the current version.

## Request / response format

- Requests and responses are **JSON** (`application/json`).
- File uploads use `multipart/form-data`.
- Timestamps are **ISO-8601** in UTC.
- IDs are strings unless the underlying type is numeric.

## Authentication

Every non-health endpoint requires `Authorization: Bearer YOUR_API_KEY`. See [Authentication](/api-reference/authentication) for details.

## Errors

Uniform error envelope:

```json
{
  "error": "Human-readable message",
  "code": "invalid_parameter",
  "details": { "field": "ccid", "issue": "not found" }
}
```

Status codes follow HTTP conventions — see [Errors](/api-reference/errors) for the full list.

## Machine-readable spec

An OpenAPI 3.1 spec is available at `https://api.bluprynt.com/openapi.json`. Feed it to your favorite generator for typed clients.
