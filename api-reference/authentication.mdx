Asset endpoints in the Bluprynt Public API require an API key on every request. Badge endpoints are public and do not require authentication.

## How to send the key

The API uses an `apiKey` security scheme. Send your key directly in the `Authorization` header — **without** a `Bearer` prefix.

<CodeGroup>

```bash curl
curl https://integrations.bluprynt.com/assets \
  -H "Authorization: YOUR_API_KEY"
```

```javascript Node.js
const res = await fetch("https://integrations.bluprynt.com/assets", {
  headers: { Authorization: process.env.BLUPRYNT_API_KEY },
});
const { data, meta } = await res.json();
```

```python Python
import os, requests

res = requests.get(
    "https://integrations.bluprynt.com/assets",
    headers={"Authorization": os.environ["BLUPRYNT_API_KEY"]},
)
res.raise_for_status()
payload = res.json()
```

</CodeGroup>

<Warning>
  The header value is the key itself — `Authorization: YOUR_API_KEY` — not `Authorization: Bearer YOUR_API_KEY`. Adding `Bearer` will fail authentication with `401`.
</Warning>

## Getting an API key

Request a key from the Bluprynt integrations team at [team@bluprynt.com](mailto:team@bluprynt.com). Include:

- The product or surface you intend to integrate with (exchange listing, wallet, dApp, explorer, etc.).
- The expected request volume per day.
- A primary contact for security and incident communication.

Keys are scoped to your organization and rate-limited per key.

## Failure modes

| Status | When | Recover by |
| --- | --- | --- |
| `401 Unauthorized` | Missing, malformed, or revoked key. | Re-check the header; rotate the key if you suspect compromise. |
| `500 Internal Server Error` | Unexpected server error. | Retry with exponential backoff; if it persists, contact support. |

The error body for `401` is:

```json
{
  "message": "Invalid API key",
  "error": "Unauthorized",
  "statusCode": 401
}
```

See the [Errors](/api-reference/errors) page for every shape and code you can expect.

## Rotating a key

To rotate, request a new key, deploy it everywhere, then ask Bluprynt to revoke the old one. Old keys remain valid until explicitly revoked — there is no automatic expiry — so it is safe to run both keys in parallel during the rollout.

<Note>
  Treat your API key like a password. Never commit it to source control, never paste it into client-side code, and rotate immediately if you believe it has leaked.
</Note>
