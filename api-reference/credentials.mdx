## Overview

A **credential** is a typed attestation — a specific piece of compliance evidence bound to an identity. Two kinds:

- **License** — authorization granted by an authority (e.g., "AMF ART authorization")
- **Disclosure** — a filed regulatory disclosure (e.g., "MiCA white paper, v1.2")

Credentials are the primitives consumed by **policy modules** at token-operation time — see the [Policy Engine](/architecture/policy-engine).

## `GET /v1/credentials`

List all credentials your key can see.

```bash
curl "https://api.bluprynt.com/v1/credentials?kind=license&identityCcid=bprnt_abc" \
  -H "Authorization: Bearer YOUR_KEY"
```

Query params:
- `kind` — `license` | `disclosure`
- `identityCcid` — filter by CCID
- `credentialType` — filter by credential type (e.g., `mica-art-authorization`)
- `status` — `active` | `revoked` | `expired`
- `limit`, `cursor` — pagination

**Response:**
```json
{
  "credentials": [
    {
      "id": "bprnt_cred_...",
      "kind": "license",
      "type": "mica-art-authorization",
      "identityCcid": "bprnt_abc123",
      "issuedBy": "AMF",
      "issuedAt": "2024-06-01T00:00:00Z",
      "expiresAt": "2026-06-01T00:00:00Z",
      "status": "active",
      "attestationId": "bprnt_att_...",
      "ipfsCid": "Qm..."
    }
  ]
}
```

## `GET /v1/credentials/{ccid}/{credentialType}`

Fetch a specific credential for an identity.

```bash
curl https://api.bluprynt.com/v1/credentials/bprnt_abc123/mica-art-authorization \
  -H "Authorization: Bearer YOUR_KEY"
```

## `POST /v1/credentials/{kind}` (restricted)

Issue a credential. Requires `credentials:write` scope; typically reserved for Bluprynt's authority role.

```bash
curl https://api.bluprynt.com/v1/credentials/license \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "identityCcid": "bprnt_abc123",
    "type": "mica-art-authorization",
    "issuedBy": "AMF",
    "issuedAt": "2024-06-01T00:00:00Z",
    "expiresAt": "2026-06-01T00:00:00Z",
    "documentCid": "Qm..."
  }'
```

## `GET /v1/credential-types`

List every credential type Bluprynt supports.

```json
{
  "credentialTypes": [
    { "code": "mica-art-authorization", "kind": "license", "description": "MiCA Asset-Referenced Token authorization." },
    { "code": "mica-emt-authorization", "kind": "license", "description": "MiCA E-Money Token authorization." },
    { "code": "regd-506b", "kind": "license", "description": "US Regulation D Rule 506(b) exemption." },
    { "code": "mica-whitepaper", "kind": "disclosure", "description": "Published MiCA white paper." }
  ]
}
```

## `POST /v1/credential-types` (restricted)

Register a new credential type. Reserved for Bluprynt staff.

## Credentials in policy modules

On-chain policy modules read credentials by querying the Bluprynt identity registry. The credential's `attestationId` points to the underlying on-chain attestation — policy modules verify the attestation's presence and validity before permitting a token operation.
