The Bluprynt Public API returns conventional HTTP status codes and a structured JSON body for every error.

## Error envelope

```json
{
  "message": "Invalid API key",
  "error": "Unauthorized",
  "statusCode": 401
}
```

| Field | Type | Description |
| --- | --- | --- |
| `message` | string | Human-readable explanation of what went wrong. |
| `error` | string | Short error label (e.g. `Unauthorized`, `Internal Server Error`). |
| `statusCode` | number | Mirrors the HTTP status code. |

## Status codes

<CardGroup cols={2}>
  <Card title="200 OK" icon="circle-check">
    The request succeeded. Asset endpoints return JSON; badge endpoints return binary image data.
  </Card>
  <Card title="400 Bad Request" icon="triangle-exclamation">
    A path or query parameter was invalid — for example, an unsupported `credential_type` or an unknown query value on the badge endpoint.
  </Card>
  <Card title="401 Unauthorized" icon="key">
    The API key is missing, malformed, or revoked. See [Authentication](/api-reference/authentication).
  </Card>
  <Card title="404 Not Found" icon="magnifying-glass">
    The asset does not exist or has not been KYI-verified. Returned by the asset-by-address and CAIP-10 endpoints.
  </Card>
  <Card title="415 Unsupported Media Type" icon="ban">
    Specific to the badge endpoint — currently returned when `format=webp` is requested (WebP is not yet available).
  </Card>
  <Card title="500 Internal Server Error" icon="server">
    Unexpected server error. Retry with exponential backoff; if it persists, contact [team@bluprynt.com](mailto:team@bluprynt.com).
  </Card>
</CardGroup>

## Endpoint-specific behavior

- `404` is only emitted by single-asset lookups (`/assets/{chain}/{account_address}` and `/assets/{namespace}/{reference}/{account_address}`). The list endpoint returns `200` with an empty `data` array when no assets match.
- `400` and `415` only appear on the badge endpoint.
- Asset endpoints never return badge-specific codes (`400`, `415`).

## Handling errors in code

```typescript TypeScript
const res = await fetch(`https://integrations.bluprynt.com/assets/${chain}/${address}`, {
  headers: { Authorization: process.env.BLUPRYNT_API_KEY! },
});

if (res.status === 404) {
  // Treat the asset as unverified — no Bluprynt KYI record exists.
  return null;
}
if (res.status === 401) {
  throw new Error("Bluprynt API key is invalid or revoked");
}
if (!res.ok) {
  const body = await res.json().catch(() => ({}));
  throw new Error(`Bluprynt API ${res.status}: ${body.message ?? res.statusText}`);
}

return res.json();
```

<Note>
  For every non-`200` response from an asset endpoint, the body matches the envelope above. For badge errors (`400`, `415`), the body may be empty — branch on status code, not body shape.
</Note>
